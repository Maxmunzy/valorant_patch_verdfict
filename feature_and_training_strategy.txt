Patch Verdict — 피처 및 학습 전략 정리
마지막 업데이트: 2026-04-02
================================================


[구조 요약]
------------------------------------------------
한 행 = 요원 1명 × 패치 이벤트 1개

입력 피처: 패치 내용 + 패치 전후 랭크 지표 + 패치 전후 VCT 지표
출력 타겟: 패치 후 VCT 픽률 변화율 (연속값) 또는 판정 레이블 (분류값)

핵심 전환:
  구버전 → 랭크 픽률·승률 변화가 메인 타겟
  신버전 → VCT 픽률 변화가 메인 타겟, 랭크 지표는 조기 신호 피처


================================================
[1] 학습 피처 (입력)
================================================

── A. 패치 내용 피처 (원인) ──────────────────────

patch_version          패치 번호 (예: 12.05)
agent                  요원 영문명
direction              nerf / buff / rework
skill_key              C / Q / E / X
change_type            damage / duration / charges / range /
                       cooldown / cost / mechanic /
                       bugfix_nerf / bugfix_buff / rework
                       (* mechanic: 즉시사용→장착 같은 메커니즘 변경)
value_before           변경 전 수치
value_after            변경 후 수치
value_change_ratio     (after - before) / before  ← 수치 변화 비율
change_reason          Riot 공식 너프/버프 근거 문장 (패치 노트에서 추출)
trigger_type           너프 트리거 유형 분류 (아래 참고)
has_bugfix             해당 패치에 실질적 버그픽스 동반 여부 (0/1)
map_change_flag        해당 패치에 맵 변경 동반 여부 (0/1)

trigger_type 분류:
  rank_stat      픽률·승률 초과형  (랭크 지표가 기준)
  pro_dominance  프로씬 지배형     (VCT 픽률이 기준)
  map_anchor     맵 고착형         (특정 맵 의존도 기준)
  skill_ceiling  숙련도 천장형     (고숙련·프로 퍼포먼스 기준)
  role_invasion  포지션 잠식형     (타 포지션 픽률 잠식 기준)


── B. 패치 전 랭크 지표 피처 (조기 신호) ──────────

pre_pr              패치 직전 3액트 평균 픽률 (Diamond+)
pre_wr              패치 직전 3액트 평균 승률 (Diamond+)
pre_pr_radiant      래디언트 픽률  ← 고티어 조기 신호
pre_pr_immortal     이모탈 픽률
pre_wr_radiant      래디언트 승률
pre_wr_vs_avg       전체 요원 평균 대비 승률 격차
pre_pr_trend        직전 3액트 픽률 추세 (상승/하락/유지)
pre_kd              패치 직전 KD 비율


── C. 패치 전 VCT 지표 피처 ──────────────────────

pre_vct_pr              패치 직전 VCT 대회 픽률
pre_vct_wr              패치 직전 VCT 대회 승률
pre_vct_maps_count      픽된 맵 수 (전 맵 vs 특정 맵 쏠림)
pre_vct_top_player_delta
                        해당 요원 최고 퍼포먼스 선수와
                        평균 선수의 스탯 격차
                        (숙련도 천장형 감지용 — 담비 같은 케이스)
pre_vct_ban_rate        VCT에서 해당 요원 주력 맵의 밴 비율
                        (map_anchor 트리거 감지용)


── D. 맥락 피처 ──────────────────────────────────

map_dependency_score  특정 맵 픽률 / 전체 맵 평균 픽률
                      (바이퍼×브리즈 같은 맵 고착 감지)
top_map_in_rotation   의존도 높은 맵이 현재 로테이션에 있는지 (0/1)
                      → yoru_map_act_history no_picks 맵 = 로테 밖
effective_map_dep     map_dependency_score × top_map_in_rotation
rival_pr_delta        동일 포지션 경쟁 요원 픽률 변화 (반사이익 감지)

pre_comp_wr           해당 요원 포함 팀 컴프 승률 (VCT 기준)
pre_comp_pick_rate    해당 요원이 포함된 컴프 비율
                      (특정 컴프에서만 채택되는지 감지)


── E. 버프 진단 피처 ──────────────────────────────
(약한 요원 분석 시 추가 활용)

wr_below_avg          승률이 전체 평균 이하인 액트 수
pr_decline_streak     픽률 하락이 연속된 액트 수
pr_vs_role_avg        동일 포지션 평균 픽률 대비 편차
vct_absence_streak    VCT에서 픽되지 않은 대회 수
last_buff_elapsed     마지막 버프 이후 경과 액트 수
nerf_buff_ratio       누적 너프 횟수 / 버프 횟수
has_rework_history    리워크 이력 여부 (0/1)
pr_radiant_gap        래디언트 픽률 - 다이아 픽률
                      (양수면 고티어에서만 쓰임 → 진입장벽형)
vct_only              VCT에서는 쓰이는데 랭크에서 안 쓰임 (0/1)
rank_only             랭크에서는 쓰이는데 VCT에서 안 쓰임 (0/1)


── 보류된 피처 ────────────────────────────────────

공수 진형별 피처:
  발로란트는 게임 시작 전 요원을 고정 선택하므로
  공격/수비에 따른 요원 픽 변화가 없음.
  맵 의존도 피처로 흡수 가능하여 별도 추가 불필요.

저격총 픽 비율:
  생존기/이동기 보유 요원에게만 의미 있는 지표.
  전체 모델에 넣으면 노이즈. 고도화 단계에서 검토.

대회 권역별 라벨링:
  발로란트는 지역 메타 편차가 타 게임 대비 작음.
  권역별로 나누면 샘플이 줄어 오히려 노이즈 증가.

2부 대회 피처:
  메타 이해도 신뢰성 문제. 1부 VCT 데이터 충분히
  확보 후 보완용으로 검토.

각 팀별 맵별 요원 조합:
  고도화 단계 피처. 수집 난이도 높고 케이스별 의미
  달라짐. 1단계 완성 후 추가.


================================================
[2] 판정 타겟 (출력)
================================================

메인 타겟 — VCT 픽률 변화율
  post_vct_pr_delta
    = (패치 후 첫 VCT 대회 픽률 - 패치 전 VCT 픽률)
      / 패치 전 VCT 픽률

이 값을 기준으로 판정 레이블 자동 생성:
  +20% 이상 유지   → INEFFECTIVE (너프가 효과 없음)
  -20% ~ +20%      → BALANCED    (의도한 조정)
  -20% ~ -60%      → EFFECTIVE   (방향은 맞음)
  -60% 이상 폭락   → EXCESSIVE   (과한 너프)
  (버프의 경우 방향 반전)

보조 타겟 — 랭크 지표 변화 (VCT 데이터 없을 때 잠정 판정용)
  post_pr_delta       패치 후 랭크 픽률 변화율
  post_wr_delta       패치 후 랭크 승률 변화량
  post_radiant_delta  래디언트 픽률 변화율 (조기 신호)

판정 타임라인:
  패치 직후       → 고티어 랭크 픽률 변화 (조기 신호, 잠정 판정)
  1~2액트 경과    → 전체 랭크 픽률·승률 안정화
  다음 VCT 대회   → 최종 판정 확정

축 2 현재 강도 판정:
  STILL OP     — 너프 이후에도 지배적
  STRONG       — 강하지만 이전보다 약해짐
  BALANCED     — 평균 수준, 최적 밸런스
  WEAKENED     — 평균 이하
  NEEDS BUFF   — 수치 조정으로 해결 가능
  NEEDS REWORK — 구조적 문제, 리워크 필요
                 (버프 진단 피처 복수 해당 시)
  PENDING      — 샘플 부족, 판정 보류

================================================
[5] 학습 전략
================================================

Step 1. 데이터 수집
  - maxmunzy 깃허브 데이터 전처리
    (diamond1~3 + immortal + radiant 필터 → 다이아+ 합산)
  - 패치 노트 크롤러 + Claude API 정형화
    (change_reason, trigger_type, mechanic 유형 포함)
  - 전체 요원 랭크 히스토리 크롤러 배치 실행
    (crawl_tracker.py UUID 교체)
  - VCT 대회별 픽률·컴프 수집
    (santiago_yoru.py 구조를 전 대회로 확장)

Step 2. 피처 엔지니어링
  - 맵 의존도 지수 계산
    (yoru_map_act_history no_picks → 로테이션 자동 추출)
  - VCT 픽률 변화율 계산 (메인 타겟)
  - 고티어 랭크 조기 신호 피처 계산
  - 컴프 승률·픽률 계산 (VCT 컴프 데이터)
  - 밴픽 데이터에서 맵 밴 비율 추출

Step 3. 모델 선택
  케이스 20개 미만    → Rule-based 보조 활용
  케이스 20~30개      → RandomForest / XGBoost
  케이스 30개 이상    → LSTM 등 시계열 모델 고려

  trigger_type별 서브모델 분리 고려:
    rank_stat + role_invasion  → 랭크 피처 위주
    pro_dominance + skill_ceiling → VCT 피처 위주
    map_anchor → 맵 의존도 피처 위주

Step 4. 검증
  - 요루 12.05 테스트셋으로 분리
  - VCT Stage 1 2026 데이터 나오면 최종 판정 비교
  - trigger_type별 정확도 따로 측정

Step 5. 실시간 검증
  - 네온 너프 발생 시 즉시 예측
  - 다음 VCT 대회 결과와 비교
