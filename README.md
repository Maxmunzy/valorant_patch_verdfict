# valorant_patch_verdict

## 🎯 Patch Verdict

> **발로란트 패치 정당성 분석 및 예측 시스템**  
> "이 너프는 맞았나?" — 데이터가 판단한다.

---

## 💡 아이디어 배경

발로란트에서 요원이 너프/버프를 받을 때마다 커뮤니티는 "이거 맞냐 틀리냐"로 감정적인 논쟁이 벌어진다.  
tracker.gg, blitz.gg, vlr.gg 같은 기존 사이트들은 **현재 픽률·승률**은 보여주지만, **패치 전후 비교**나 **"이 패치가 맞았나"에 대한 판단**은 해주지 않는다.

Patch Verdict는 그 공백을 채우려는 시도다.  
감정이 아닌 **데이터**로 패치의 정당성을 분석하고, 나아가 **다음에 너프받을 요원이 누구인지 사전에 예측**하는 것까지 목표로 한다.

## 🎯 샘플링 기준

- **서버**: 전 지역 합산 (Global)
  - vstats.gg가 지역 필터를 지원하지 않아 전 지역 합산으로 수집
  - 발로란트는 맵 로테이션·패치가 전 서버 동시 적용되고 요원 풀이 적어 지역별 메타 편차가 타 게임 대비 작음 → 전 지역 합산이 오히려 샘플 안정성 측면에서 합리적
- **티어**: 다이아몬드 이상 (Diamond+)
  - 그 이하 티어는 메타 이해도보다 선호도로 픽하는 경우가 많아 노이즈가 큼
- **데이터 범위**: E2A1 이후 (maxmunzy), E6A3 이후 (vstats.gg)

## ⚖️ 판정 체계

Patch Verdict는 하나의 패치에 대해 **두 가지 질문**을 동시에 답한다.

| 축 | 질문 | 기준 |
|---|---|---|
| **패치 정당성** | 이 패치는 맞았나? | 패치 전후 VCT 픽률 변화 |
| **현재 강도** | 지금 이 요원 써도 되나? | 현재 랭크 승률·픽률 |

---

### 축 1 — 패치 정당성 (Patch Verdict)

**메인 판정 기준: 패치 전후 VCT 픽률 변화율**

프로 선수들은 돈이 걸린 경쟁에서 가장 합리적인 선택을 한다.  
Riot의 의도가 아닌 **프로들의 실제 반응**이 판정 기준이다.

```
+20% 이상 유지   →  INEFFECTIVE — 너프가 메타에 영향을 주지 못함
-20% ~ +20%      →  BALANCED    — 의도한 방향으로 정확히 조정됨
-20% ~ -60%      →  EFFECTIVE   — 방향은 맞지만 강도가 다소 강하거나 약함
-60% 이상 폭락   →  EXCESSIVE   — 강도가 지나쳐 메타를 왜곡함
```

**보조 신호: 고티어 랭크 픽률 (조기 감지)**

VCT 대회 전, 래디언트·이모탈 픽률 변화가 선행 지표 역할을 한다.

```
판정 타임라인:
패치 직후       → 고티어 랭크 픽률 변화 (조기 신호, 잠정 판정)
1~2액트 경과    → 전체 랭크 픽률·승률 안정화
다음 VCT 대회   → 최종 판정 확정
```

---

### 축 2 — 현재 강도 (Current State)

```
STILL OP      — 너프/버프 이후에도 여전히 지배적
STRONG        — 강하긴 하지만 패치 전보다 약해짐
BALANCED      — 승률·픽률 모두 평균 수준, 최적 밸런스
WEAKENED      — 버프까진 아니지만 평균 이하로 내려앉음
NEEDS BUFF    — 수치 조정으로 해결 가능
NEEDS REWORK  — 구조적 문제, 리워크 필요
PENDING       — 샘플 부족, 판정 보류
```

| 단계 | 승률 | 픽률 순위 |
|---|---|---|
| STILL OP | 51%+ | 상위 10위 이내 |
| STRONG | 49~51% | 상위 20위 이내 |
| BALANCED | 47~49% | 중위권 |
| WEAKENED | 45~47% | 하위권 |
| NEEDS BUFF | 45% 미만 | 하위권 |

## 🔍 너프 트리거 유형 분류

패치 노트의 공식 근거 문장을 분석해 유형을 분류하고, 유형별로 판정 피처를 달리 적용한다.

| 유형 | 설명 | 핵심 판정 피처 | 사례 |
|---|---|---|---|
| rank_stat | 랭크 픽률·승률 초과 | 랭크 지표 변화 | 일반적 밸런스 조정 |
| pro_dominance | 프로씬 필수픽화 | VCT 픽률 변화 | 체임버, 제트 |
| role_invasion | 타 포지션 입지 잠식 | 경쟁 요원 픽률 반사이익 | 요루 12.05 |
| map_anchor | 특정 맵 고착 | 맵 의존도 지수 + VCT 밴율 | 바이퍼 |
| skill_ceiling | 고숙련 천장 과도 | VCT 퍼포먼스 편차 | 네온 (예상) |

복수 트리거 케이스가 일반적이며, 트리거가 겹칠수록 너프 강도가 세진다.

## 🧠 모델 구조

### 1단계 — 사후 판정 모델 (Post-patch Verdict)

**입력 피처**

_패치 내용 (원인)_
- 변경된 스킬, 수치 변화 비율, 방향 (nerf/buff/rework)
- change_type: damage / duration / charges / mechanic 등
- Riot 공식 너프 근거 → 트리거 유형 분류 (trigger_type)

_패치 전 랭크 지표 (조기 신호)_
- Diamond+ 픽률·승률 (직전 3액트 평균)
- 래디언트·이모탈 픽률 (고티어 선행 신호)
- KD 비율, 픽률 추세

_패치 전 VCT 지표_
- 직전 대회 픽률·승률
- 맵별 픽률 편중도, 밴율
- 최고 퍼포먼스 선수와 평균 선수의 스탯 격차 (숙련도 천장 감지)
- 해당 요원 포함 컴프 승률·픽률

_맥락 피처_
- 맵 의존도 지수 × 해당 맵 로테이션 여부
- 동일 포지션 경쟁 요원 픽률 (반사이익 감지)

**출력**
- 패치 정당성: BALANCED / EFFECTIVE / EXCESSIVE / INEFFECTIVE
- 현재 강도: STILL OP / STRONG / BALANCED / WEAKENED / NEEDS BUFF / NEEDS REWORK

**레이블 생성**
- 패치 후 첫 VCT 대회 픽률 변화율로 자동 생성

---

### 2단계 — 사전 예측 모델 (Pre-patch Prediction)

1단계 모델의 부산물로 자연스럽게 파생.

**출력 예시**
```
"다음 너프받을 요원은 89% 확률로 네온입니다.
이유: skill_ceiling + pro_dominance 트리거 동시 감지"
```
