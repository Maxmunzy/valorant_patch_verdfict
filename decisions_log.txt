Patch Verdict — 결정 배경 메모
마지막 업데이트: 2026-04-02
================================================


[1] 데이터 수집 범위: 전체 오픈부터 X, E6A3부터 O
------------------------------------------------
이유: vstats.gg 히스토리 데이터 자체가 E6A3부터만 존재.
패치 내용은 수집해도 대응하는 픽률·승률 데이터가 없으면 학습에 못 씀.

단, maxmunzy/valorant-agent-stats 깃허브 데이터가
E2A1~E8A2 구간의 전 요원 픽률·승률·KD·매치수를 티어별로 수록.
diamond1~3 + immortal + radiant 필터링 시 다이아+ 기준으로 활용 가능.
→ 체임버(E5), 제트(E4) 등 대너프 케이스 랭크 데이터 확보 가능.
→ 다만 VCT 데이터는 별도 수집 필요.


[2] 전 지역 합산 데이터의 타당성
------------------------------------------------
vstats.gg 지역 필터 미지원으로 전 지역 합산 수집.
발로란트 특성상 오히려 합리적인 선택임.

근거:
- 맵 로테이션이 전 서버 동시 적용 → 메타 형성 조건 자체가 동일
- 요원 풀이 25명 내외로 적어서 지역별 메타 수렴이 빠름
- VCT 구조가 인터내셔널 대회 중심 → 지역 간 스크림으로 메타 공유 활발
- 패치 전 서버 동시 적용 → 지역별 메타 발견 시간차 없음

결론: AP 서버만 고집했으면 샘플이 적어 오히려 노이즈가 컸을 수 있음.
전 지역 합산이 단순한 차선책이 아니라 합리적인 기준.


[3] 초기 모델의 근본 문제 — 원인 피처 없음
------------------------------------------------
처음에 픽률·승률(결과) 위주로 크롤링 방향을 잡아서
패치 내용(원인)이 빠진 구조가 됐음.

올바른 ML 순서: 패치 내용(원인) → 픽률·승률 변화(결과) → 판정
실제로 한 순서: 픽률·승률 변화(결과) → 판정
→ 원인이 없으니 일반화 불가, 요루 케이스에서만 작동하는 구조


[4] 핵심 방향 전환 — 랭크 데이터 메인 → VCT 데이터 메인
------------------------------------------------
배경:
Riot이 패치를 결정하는 근거가 "랭크 픽률·승률이 높아서"가 아닌 경우가 많음.

요루  → 프로씬 포지션 잠식 (랭크 지표는 평범)
바이퍼 → 특정 맵 고착으로 다양성 훼손
네온  → 고숙련 천장 과도 (랭크 픽률·승률은 평범)
체임버 → 프로씬 필수픽 구조

전환 결과:
  구버전 → 랭크 픽률·승률 변화가 메인 판정 기준
  신버전 → VCT 픽률 변화가 메인 판정 기준
           랭크 고티어 데이터는 조기 신호 피처로 역할 전환

근거: 프로 선수들은 돈이 걸린 경쟁에서 가장 합리적인 선택을 함.
VCT 픽률이 메타를 가장 정확하게 반영.
Riot 의도가 아닌 프로들의 실제 반응이 판정 기준.


[5] 너프 트리거 유형 분류 도입
------------------------------------------------
Riot 패치 노트가 항상 공식 근거 문장을 포함함.
→ Claude API로 근거 문장 추출 + trigger_type 1차 분류 + 사람 검토

유형 분류:
  rank_stat      픽률·승률 초과형
  pro_dominance  프로씬 지배형
  role_invasion  포지션 잠식형
  map_anchor     맵 고착형
  skill_ceiling  숙련도 천장형

복수 트리거 케이스가 일반적임:
  요루 12.05 → role_invasion + pro_dominance 동시
  네온 (예상) → skill_ceiling + pro_dominance 동시
→ 트리거가 겹칠수록 너프 강도가 세지는 패턴을 학습 가능
→ 케이스 수 × 트리거 조합 수가 실질적인 학습 데이터양


[6] change_type에 mechanic 추가
------------------------------------------------
웨이레이 12.06 패치 분석 중 발견.
"포화가 즉시 사용에서 장착으로 변경" 같이
수치 변경이 아닌 메커니즘 자체가 바뀌는 패치가 존재함.

기존 change_type에 mechanic 카테고리 추가:
  mechanic: 즉시사용→장착, 패시브→액티브 등 작동 방식 변경
  value_before/after는 null, description으로 내용 기술


[7] 버프 진단 피처 도입
------------------------------------------------
배경: 너프 판정만이 아니라 버프 방향 제안까지 다뤄야 함.
"왜 안 쓰이는지"의 원인이 다양하고 원인마다 버프 방향이 달라짐.

[스킬 자체 문제]
- 픽률 낮고 승률도 낮음 → 스킬이 약함
- 동일 포지션 대비 승률 격차
- 스킬별 사용률

[대체재 문제]
- 동일 포지션 경쟁 요원 픽률이 압도적으로 높음
- 해당 요원 픽률 하락과 경쟁 요원 픽률 상승이 반비례

[맵 의존 문제]
- 특정 맵에서만 픽률 높음 → 범용성 부족
- 로테이션 맵 변경 시 픽률 급락

[진입장벽 문제]
- 픽률 낮은데 승률은 높음 → 잘 하는 사람만 씀
- 고티어일수록 픽률 높아지는 패턴 (pr_radiant_gap)
- VCT에서는 쓰이는데 랭크에서 안 쓰임

[메타 구조 문제]
- 현재 메타에서 해당 포지션 자체 수요가 없음
- VCT 컴프 데이터에서 해당 요원 포함 컴프 승률
- 팀 컴프 구성상 채택되기 어려운 구조

[시간 흐름 문제]
- 픽률이 꾸준히 하락 추세
- 출시 초반 픽률 대비 현재 픽률 격차
- 리워크 이후 픽률 회복 여부

[패치 이력]
- 마지막 버프 이후 경과 액트 수
- 누적 너프 횟수 대비 버프 횟수
- 리워크 이력 여부


[8] NEEDS REWORK 판정 추가
------------------------------------------------
버프 진단 피처들이 복수로 동시에 켜지는 경우
수치 조정으로 해결 안 되는 구조적 문제로 판단.

판정 체계 확장:
  STILL OP     — 너프 이후에도 지배적
  STRONG       — 강하지만 이전보다 약해짐
  BALANCED     — 평균 수준, 최적 밸런스
  WEAKENED     — 평균 이하
  NEEDS BUFF   — 수치 조정으로 해결 가능
  NEEDS REWORK — 구조적 문제, 리워크 필요
  PENDING      — 샘플 부족, 판정 보류

NEEDS REWORK 트리거 조건 (복수 해당 시):
- 픽률 낮고 승률도 낮음
- 대체재 존재
- VCT에서도 안 쓰임
- 오래 방치됨
- 너프만 계속 맞음
→ 하버가 교과서 케이스


[9] 보류된 피처들
------------------------------------------------
공수 진형별 피처:
  발로란트는 게임 시작 전 요원을 고정 선택하므로
  공격/수비에 따른 요원 픽 변화가 없음.
  맵 의존도 피처로 흡수 가능하여 별도 추가 불필요.

저격총 픽 비율:
  생존기/이동기 보유 요원에게만 의미 있는 지표.
  전체 모델에 넣으면 노이즈. 고도화 단계에서 검토.

대회 권역별 라벨링:
  발로란트는 지역 메타 편차가 타 게임 대비 작음.
  권역별로 나누면 샘플이 줄어 오히려 노이즈 증가.

2부 대회 피처:
  메타 이해도 신뢰성 문제. 1부 VCT 데이터 충분히
  확보 후 보완용으로 검토.

각 팀별 맵별 요원 조합:
  고도화 단계 피처. 수집 난이도 높음.
  1단계 완성 후 추가.


[10] 2단계 예측 모델은 1단계에서 자동으로 파생
------------------------------------------------

현재 요원 데이터를 넣었을 때
"어떤 유형으로 너프/버프받을 가능성이 있는가"까지
자연스럽게 나오는 구조.

별도 모델이 아니라 1단계 완성의 부산물로 얻어짐.
출력: "다음 너프받을 요원은 X% 확률로 네온입니다. 이유는..."

[12] 커뮤니티 토론 플랫폼 기능
------------------------------------------------
Patch Verdict의 부가 가치로 커뮤니티 토론의 질을 높일 수 있음.

기존: 감정 기반 논쟁 → 결론 없이 끝남
도입 후: 데이터 기반 공통 기준점 생성
→ 동의하든 반박하든 논거의 질이 달라짐

수익 모델:
- 광고 모델 (패치 때마다 트래픽 폭발)
- 구독 모델 (상세 분석 리포트)
- tracker.gg / blitz.gg 탭 통합
- Riot Games 밸런스팀 대상 B2B
